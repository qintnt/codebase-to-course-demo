What Is This Thing?

A skill that reads code and writes a beautiful, interactive course — so anyone can understand how software works.

Imagine This

You find a cool project on GitHub. It works. But looking at the code feels like staring at a foreign language.

This skill fixes that. It reads the codebase, figures out how everything connects, and writes a beautiful interactive course — complete with animations, quizzes, and plain-English code explanations.

💡
Key Insight

This is meta — the course you're looking at right now was built by this very skill. It reads its own source code and taught you how it works. 🤯

Who It's For

The target audience is "vibe coders" — people who build software by telling AI tools what they want, without a traditional computer science background.

Design Philosophy

Warm, visual, metaphor-rich. Everything designed for someone who's never opened a code file.

Build First, Understand Later

This skill inverts traditional education. Instead of years of theory before touching real code, you start with something that already works — then peel back the layers.

🏗️
The Metaphor

Think of it like taking apart a watch. You already know it tells time. Now you open it up to see the gears — each one makes sense in context because you know what the whole thing does.

❌

Old Way
Memorize concepts → maybe build something → most people quit

vs
✅

This Way
Build something → see it work → now understand the how

Show, Don't Tell

Every screen is at least 50% visual. Text blocks are limited to 2-3 sentences. If something can be a diagram, animation, or interactive element — it shouldn't be a paragraph.

The Rule 
// ❌ BAD — wall of text
The extension consists of multiple
files including a manifest, content
script, service worker, popup HTML...
// (20 more lines of explanation)

// ✅ GOOD — use a file tree
Use visual file tree
with descriptions
instead of paragraphs
WHY THIS MATTERS

Dense paragraphs overwhelm non-technical readers. They look like homework.

Visual structures like file trees let readers scan and understand in seconds.

The gotchas.md file literally lists "Walls of Text" as the #1 mistake to avoid.

Design Identity

The visual style has strong opinions — no generic purple gradients:

🌅

Warm Palette

Off-white backgrounds like aged paper, warm grays. Feels like a developer's notebook.

✏️

Distinctive Type

Bricolage Grotesque for headings — bold and geometric. Never Inter or Roboto.

🎯

One Accent Color

Vermillion, coral, teal, amber, or forest. One confident accent, not a rainbow.

💻

Dark Code Blocks

Catppuccin-inspired syntax highlighting on deep indigo-charcoal. Feels like a real IDE.

No Recycled Metaphors

Each concept gets a unique metaphor that fits organically. The "restaurant" metaphor is banned — it's been overused to death.

Database → A library with a card catalog system
Authentication → A bouncer checking IDs at the door
Rate Limiting → A nightclub with a capacity limit
Cache → Keeping leftovers instead of cooking every meal from scratch

Quizzes Test Doing, Not Knowing

No "What does API stand for?" Instead:

Real Quiz Style

A user reports stale data after switching pages. Where would you look first?

⚠️
Original Code Only

Code snippets are never modified or simplified. The learner should be able to open the actual file and see the exact same code they learned from. This builds real confidence.

05

The Interactive Engine

A single 14KB JavaScript file powers every quiz, animation, and interaction. No frameworks, no dependencies.

One File Does Everything

The entire interactive experience is driven by main.js. It doesn't use React, Vue, or any framework. Just vanilla JavaScript with smart auto-detection:

📍

Auto-Detect

Scans the page for class names like .chat-window, .flow-animation, .quiz-container — and activates the right engine for each.

🎬

Scroll Animations

IntersectionObserver reveals content as you scroll down, with staggered children animations.

📖

Glossary Tooltips

position: fixed tooltips appended to document.body so they're never clipped by overflow containers.

📱

Mobile Support

Touch events for drag-and-drop, responsive grid layouts, mobile-first design throughout.

How Auto-Detection Works

The skill uses querySelectorAll to find components by their class name and wire them up automatically:

main.js 
// Find every chat window
$$('.chat-window')
  .forEach(el => initChat(el));

// Find every flow animation
$$('.flow-animation')
  .forEach(el => initFlow(el));

// Find every drag-and-drop
$$('.dnd-container')
  .forEach(el => initDnD(el));
PLAIN ENGLISH

Find every element with the "chat window" class...

...and run the chat engine on each one.

Find every flow animation element...

...and initialize the flow animation engine.

Same pattern for drag-and-drop containers.

Never Regenerate CSS or JS

One of the most important rules in the skill:

🔒
Critical Rule

The AI never regenerates styles.css or main.js. It copies them verbatim from references/. Its only job is to write the module HTML content. This keeps the design consistent and the code working.
