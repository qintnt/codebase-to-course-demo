Phase 1: Analyze the Codebase

Before writing anything, the skill deeply reads the target codebase. It's like a detective studying a crime scene:

🕵️
Identify the Actors

What are the main components, services, and modules?

🔗
Trace Data Flows

How does information move between parts?

🗺️
Map the User Journey

What happens end-to-end when someone uses the app?

🧩
Find the Clever Bits

Caching, lazy loading, error handling — the patterns worth teaching

🤖
AI Behavior

The skill figures out what the app does by reading the README and entry points. It doesn't ask the user — they may not know either!

Phase 3: Build — Sequential vs Parallel

Here's where it gets clever. The skill chooses its strategy based on complexity:

SKILL.md 
Simple codebase → sequential
  Write modules one at a time

Complex codebase → parallel
  Write module briefs first
  Dispatch to sub-agents
  Batch of up to 3 at once
PLAIN ENGLISH

Small, simple projects get the straightforward approach...

The AI writes each module in order, checking quality as it goes.

Big, complex projects trigger the smart approach...

Write briefs for each module first, so writing agents never need to re-read the whole codebase.

Then split the work across multiple AI sub-agents — up to 3 at the same time.
