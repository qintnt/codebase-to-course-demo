Design Philosophy

Warm, visual, metaphor-rich. Everything designed for someone who's never opened a code file.

Build First, Understand Later

This skill inverts traditional education. Instead of years of theory before touching real code, you start with something that already works — then peel back the layers.

🏗️
The Metaphor

Think of it like taking apart a watch. You already know it tells time. Now you open it up to see the gears — each one makes sense in context because you know what the whole thing does.

❌

Old Way
Memorize concepts → maybe build something → most people quit

vs
✅

This Way
Build something → see it work → now understand the how

Show, Don't Tell

Every screen is at least 50% visual. Text blocks are limited to 2-3 sentences. If something can be a diagram, animation, or interactive element — it shouldn't be a paragraph.

The Rule 
// ❌ BAD — wall of text
The extension consists of multiple
files including a manifest, content
script, service worker, popup HTML...
// (20 more lines of explanation)

// ✅ GOOD — use a file tree
Use visual file tree
with descriptions
instead of paragraphs
WHY THIS MATTERS

Dense paragraphs overwhelm non-technical readers. They look like homework.

Visual structures like file trees let readers scan and understand in seconds.

The gotchas.md file literally lists "Walls of Text" as the #1 mistake to avoid.

Design Identity

The visual style has strong opinions — no generic purple gradients:

🌅

Warm Palette

Off-white backgrounds like aged paper, warm grays. Feels like a developer's notebook.

✏️

Distinctive Type

Bricolage Grotesque for headings — bold and geometric. Never Inter or Roboto.

🎯

One Accent Color

Vermillion, coral, teal, amber, or forest. One confident accent, not a rainbow.

💻

Dark Code Blocks

Catppuccin-inspired syntax highlighting on deep indigo-charcoal. Feels like a real IDE.

No Recycled Metaphors

Each concept gets a unique metaphor that fits organically. The "restaurant" metaphor is banned — it's been overused to death.

Database → A library with a card catalog system
Authentication → A bouncer checking IDs at the door
Rate Limiting → A nightclub with a capacity limit
Cache → Keeping leftovers instead of cooking every meal from scratch

Quizzes Test Doing, Not Knowing

No "What does API stand for?" Instead:

Real Quiz Style

A user reports stale data after switching pages. Where would you look first?

⚠️
Original Code Only

Code snippets are never modified or simplified. The learner should be able to open the actual file and see the exact same code they learned from. This builds real confidence.
