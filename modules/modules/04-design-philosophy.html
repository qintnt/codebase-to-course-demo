设计哲学

温暖的视觉、丰富的比喻、通俗易懂。一切都是为从没看过代码的人设计的。

先做，再理解

这个技能颠覆了传统教育。不是先学几年理论再碰代码，而是从一个已经能跑的东西开始——然后一层层剥开看。

🏗️
比喻

就像拆开一块手表。你已经知道它能看时间，现在打开表盖看齿轮——每一块都在上下文里有意义，因为你知道整体是干什么的。

❌

旧方式
背概念 → 也许能做出东西 → 大多数人放弃了

vs
✅

这个技能
先做出东西 → 看到它跑 → 然后理解原理

展示，不要说教

每个页面至少 50% 是视觉内容。文字块最多 2-3 句。能做成图、动画、交互元素的，就不要写成一整段。

这条规则 
// ❌ 错误——文字墙
这个扩展由多个文件组成，
包括清单文件、内容脚本、
Service Worker、弹窗 HTML...
//（20 行解释继续）

// ✅ 正确——用文件树
用 可视化文件树
配上 描述文字
代替 长段落
为什么重要

大段文字会让非技术读者望而生畏，看起来像作业。

文件树这样的视觉结构让读者几秒钟就能扫懂。

gotchas.md 文件里明确把"文字墙"列为头号错误。

视觉风格

视觉设计有很明确的偏好——拒绝千篇一律的紫色渐变：

🌅

暖色调

偏米色的背景像旧纸张，暖灰色。感觉像程序员的笔记本。

✏️

有个性的字体

Bricolage Grotesque 做标题——大胆有几何感。绝不用 Inter 或 Roboto。

🎯

一个强调色

朱红、珊瑚、蓝绿、琥珀、森林——只选一个自信的强调色。

💻

暗色代码块

Catppuccin 风格的语法高亮，深靛蓝色背景。感觉像真正的 IDE。

比喻绝不重复

每个概念都有一个专属比喻，自然贴切。"餐厅"比喻被永久封禁——已经被用烂了。

数据库 → 有目录卡片的图书馆
身份验证 → 门口查身份证的保安
频率限制 → 有容纳上限的夜店
缓存 → 留剩菜而不是每顿都从头做

真实考题

用户反馈切换页面后数据没更新。你会先检查哪里？

⚠️
代码必须是原始的

课程里的代码片段绝不修改或简化。学习者应该能打开真实文件，看到和课程里一模一样的代码。这能建立真正的信心。
